1. Overview

This guide walks you through setting up everything you need to develop and manage your photography portfolio site. The site is built with Next.js and hosted on Vercel, with photos stored on Cloudflare R2 and album data in a Turso database.

You'll use Claude Code to make changes to the site in plain English — no coding knowledge required.

What You Need

  • A Mac (you have this)
  • Your Claude Code subscription ($20/mo — you have this)
  • A GitHub account (free)
  • About 30 minutes for initial setup

What You'll Set Up

  • Vercel — hosts your website, auto-deploys when you push code
  • Turso — stores album and photo metadata (free tier)
  • Cloudflare — domain name, image storage (R2), and CDN
  • Claude Code — your AI assistant for making changes

2. Quick Start

There's an install script that handles all the boring stuff — installing Homebrew, Node.js, Git, Claude Code, and all the service CLIs. Open Terminal on your Mac and paste this:

The One-Liner

This installs everything and clones the project. It's safe to run more than once.

bash ~/poor-mans-flicker/setup/install.sh

Don't have the repo yet?

If this is a fresh Mac and you haven't cloned the project, run these two commands first:

# Install Homebrew + Git first
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
brew install git

# Clone the project
cd ~
git clone https://github.com/txcfi-scott/poor-mans-flicker.git

# Now run the setup script
bash ~/poor-mans-flicker/setup/install.sh

What the Script Does

  • Installs Homebrew (Mac package manager) — if not already installed
  • Installs Node.js and Git
  • Installs Claude Code globally
  • Installs service CLIs: Turso, Vercel, Wrangler (Cloudflare)
  • Clones the project repository
  • Runs npm install for project dependencies
  • Creates a .env.local template file

After the Script Finishes

You'll need to log into each service's CLI:

turso auth login      # Opens browser → log in with GitHub
vercel login          # Opens browser → log in with GitHub
wrangler login        # Opens browser → log in with Cloudflare

3. Service Setup

Each service needs a free account and a bit of configuration. Click each section below to expand the instructions.

V Vercel — Hosts Your Website

Vercel runs your website and automatically re-deploys whenever you push code to GitHub.

Sign Up

  1. Go to vercel.com/signup
  2. Click "Continue with GitHub" (use the same GitHub account that has your repo)
  3. That's it — your account is linked to GitHub automatically

Connect Your Repository

  1. Go to vercel.com/new
  2. Find "poor-mans-flicker" in the list and click "Import"
  3. Leave all settings as default
  4. Click "Deploy"
  5. Your site is now live at a .vercel.app URL

Add Environment Variables

  1. Go to your project on Vercel → Settings → Environment Variables
  2. Add each variable from the Quick Reference table below

Custom Domain (Optional)

  1. Go to your project → Settings → Domains
  2. Add your domain (e.g., chrishardingphotography.com)
  3. It will tell you what DNS records to add in Cloudflare

CLI Access (for Claude Code)

After running vercel login, Claude Code can:

vercel --prod        # Deploy to production
vercel ls              # Check deployment status
vercel logs            # View error logs
vercel env add NAME   # Add an environment variable
T Turso — Your Database

Turso stores your album and photo information (not the actual images — those go on Cloudflare R2).

Sign Up

  1. Go to turso.tech
  2. Click "Start for Free"
  3. Sign up with GitHub

Create Your Database

After running turso auth login from the Quick Start step:

# Create the database
turso db create poor-mans-flicker

# Get the connection URL (save this!)
turso db show poor-mans-flicker --url

# Create an auth token (save this too!)
turso db tokens create poor-mans-flicker

Save these values!

The URL becomes TURSO_DATABASE_URL and the token becomes TURSO_AUTH_TOKEN in your environment variables.

Free Tier Limits

You get way more than you'll ever need for a photo portfolio:

  • 9 GB total storage
  • 1 billion row reads/month
  • 25 million row writes/month

CLI Access (for Claude Code)

turso db create/destroy          # Create or delete databases
npx drizzle-kit push              # Update database schema
turso db shell poor-mans-flicker "SELECT * FROM albums"
turso db tokens create             # Manage tokens
turso db inspect poor-mans-flicker # View usage stats
C Cloudflare — Domain + Image Storage

Cloudflare handles three things: your domain name, photo file storage (R2), and fast worldwide delivery via CDN.

Sign Up

  1. Go to dash.cloudflare.com/sign-up
  2. Create account with email + password

Register Your Domain

  1. In Cloudflare dashboard → Domain Registration → Register Domains
  2. Search for your domain (e.g., chrishardingphotography.com)
  3. Purchase it (~$10.46/year)

Create an R2 Bucket (Image Storage)

  1. In Cloudflare dashboard → R2 Object Storage
  2. Click "Create bucket"
  3. Name it pmf-photos
  4. Click "Create bucket"

Set Up Public Access for the Bucket

  1. Go to your bucket → Settings
  2. Under "Public Access" → click "Allow Access"
  3. Add a custom domain: photos.chrishardingphotography.com
  4. Cloudflare will automatically set up DNS for this

Create R2 API Token

  1. Go to your bucket → Settings → "Manage R2 API Tokens"
    Or: My Profile → API Tokens → R2
  2. Click "Create API Token"
  3. Permissions: "Object Read & Write"
  4. Scope: Select your pmf-photos bucket
  5. Click "Create API Token"

Save these immediately — you won't see them again!

Access Key ID → this is your R2_ACCESS_KEY

Secret Access Key → this is your R2_SECRET_KEY

Get Your R2 Endpoint

Your R2 endpoint URL follows this pattern:

https://<ACCOUNT_ID>.r2.cloudflarestorage.com

Find your Account ID: Cloudflare dashboard → right sidebar → "Account ID"

DNS for Vercel

When you add your domain in Vercel, it gives you DNS records to add:

  1. Go to your domain → DNS → Records
  2. Add the records Vercel tells you (usually a CNAME record)
  3. Set the proxy status to "DNS only" (gray cloud icon) for Vercel records

Create a General API Token (for DNS management)

  1. My Profile → API Tokens → Create Token
  2. Use template "Edit zone DNS"
  3. Zone Resources: select your domain
  4. Create Token and save it

CLI Access (for Claude Code)

wrangler r2 object put/get/delete   # Manage files in R2
wrangler r2 bucket list              # List buckets

4. Environment Variables Quick Reference

After setting up all three services, you'll have these values. Add them to both Vercel (for production) and your local .env.local file (for development).

Variable Where It Comes From
TURSO_DATABASE_URL Turso: turso db show poor-mans-flicker --url
TURSO_AUTH_TOKEN Turso: turso db tokens create poor-mans-flicker
R2_ENDPOINT Cloudflare: https://<account-id>.r2.cloudflarestorage.com
R2_ACCESS_KEY Cloudflare: R2 API token → Access Key ID
R2_SECRET_KEY Cloudflare: R2 API token → Secret Access Key
R2_BUCKET pmf-photos (what you named the bucket)
R2_PUBLIC_URL https://photos.yourdomain.com (R2 custom domain)
ADMIN_TOKEN You make this up — any long random string
NEXT_PUBLIC_SITE_URL Your domain: https://chrishardingphotography.com

Checklist: Where to add these

  • Vercel → Project Settings → Environment Variables (for production)
  • Local .env.local file (for local development)

5. Using Claude Code

Claude Code is your AI-powered assistant. You tell it what you want in plain English, and it reads the code, understands the project, and makes the changes for you.

Starting Claude Code

cd ~/poor-mans-flicker
claude

That's it. Claude automatically reads the project's CLAUDE.md file and understands how everything works.

What to Say

Just describe what you want, like you'd tell a person:

"Add a new album called Summer 2026"

"Change the slideshow transition speed to 8 seconds"

"Fix the layout on mobile"

"Add a watermark to uploaded photos"

Deploying Changes

After Claude makes changes, push them live:

git add -A
git commit -m "description of changes"
git push

Vercel automatically deploys the update within a minute or two.

Running the Site Locally

npm run dev

Then open http://localhost:3000 in your browser to preview changes before deploying.
